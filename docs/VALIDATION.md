# Валидация стратегии (по §0.1 CONTEXT.md)

Выполнена до написания продуктового кода. Дата: 2026-07-19.

## 1. Логика стратегии

- Цепочка «AI видит транзакции → предсказывает кассовый разрыв → предлагает продукт банка»
  прямо вытекает из ядра исследования: отток клиентов Платежного бизнеса происходит из-за
  закрытия бизнеса, а не ухода к конкурентам; главная причина закрытия — нехватка знаний и
  опыта (≈ половина закрывших, «Корпорация МСП» + АСИ + МСП Банк + Frank RG, июнь 2026).
- Дифференциация артикулирована корректно: GigaChat/YandexGPT/LivePlan работают на общих
  данных, АльфаКопилот — на реальных транзакциях эквайринга конкретного клиента. Тренд
  подтвержден рынком (Revolut AIR, 2026).
- **Слабое место (говорить честно на защите):** переход «нехватка знаний» → «кассовый разрыв
  как главный симптом» данными не доказан. Формулировка для жюри: кассовый разрыв — самое
  частое финансовое проявление неопытности, а не единственная доказанная причина закрытия.
  Методология исследования «Корпорации МСП» (выборка) в открытых источниках не раскрыта —
  это уже помечено в стратегии, держать наготове.

## 2. Реалистичность демо-сценария Марины

- Сценарий правдоподобен и порождается генератором, а не рисуется: месяцы 1–2 рост
  (набор клиентской базы), месяц 3 — сезонный летний спад в бьюти плюс разовая закупка
  оборудования в рассрочку, месяц 4 — баланс положительный, но буфер сжимается ниже 30 дней.
- Прогноз «разрыв через N дней» выводится экстраполяцией реального тренда расходов/доходов
  и обязан совпадать с графиком (закреплено тестом).

## 3. Архитектура против сроков

- 4 языка + микросервисы одновременно — главный риск срыва. Решение (подтверждено
  пользователем): **static-first**. Сначала полный клиентский демо-сайт: детерминированные
  данные, TS-порт финансовой математики, мок-LLM, симуляция пошагового стриминга.
  Он же деплоится на GitHub Pages.
- Контракт пайплайна во фронтенде — поток событий (`AsyncIterable<AnalysisEvent>`),
  идентичный по форме будущему WS/SSE от Rust Gateway: бэкенды подключаются позже без
  переделки UI. TS-математика шага 1 — спецификация для будущего C++ Financial Engine
  (те же входы/выходы, те же тесты-ожидания).

## 4. Декомпозиция AI-цепочки

- 5 шагов сохранены: диагностика → прогноз разрыва → классификация риска → рекомендация →
  проактивное сообщение. По числу шагов сопоставимо с прецедентом Сбера (5 запросов).
- Уточнения: шаг 1 — чистая детерминированная математика (не LLM), шаг 3 — пороговая
  логика (>30 дней зеленый, 14–30 желтый, <14 красный), шаги 2/4/5 — LLM (в статике —
  детерминированные мок-фикстуры за интерфейсом провайдера). Это честнее показывает
  «техническую глубину»: жюри видит, где считает движок, а где рассуждает модель.
